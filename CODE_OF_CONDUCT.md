# Code of Conduct

## Our Commitment

We want `atldays/laravel-secrets` to be a professional, respectful, and constructive project for everyone who participates in it.

Contributors, maintainers, and community members are expected to help create an environment that is:

- welcoming
- respectful
- technically honest
- collaborative
- safe for open discussion and feedback

## Expected Behavior

Examples of behavior that contribute to a positive environment include:

- showing respect for different viewpoints and experience levels
- giving constructive, specific, and professional feedback
- assuming good intent while staying clear and direct
- accepting feedback gracefully
- focusing discussions on the code, the design, and the problem being solved
- helping keep the project useful for maintainers, contributors, and users

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment, abuse, or intimidation
- personal attacks, insults, or derogatory language
- discrimination or exclusionary behavior
- trolling, inflammatory comments, or intentionally disruptive conduct
- publishing private information about others without permission
- hostile behavior in issues, pull requests, code review, or other project spaces

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub issues
- pull requests
- code review discussions
- commit and release discussions
- any public or private communication directly related to the project

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct and may take any action they consider appropriate in response to unacceptable behavior.

This may include:

- requesting a change in behavior
- moderating discussions
- closing or locking conversations
- rejecting contributions
- temporarily or permanently restricting participation

## Reporting

If you experience or witness unacceptable behavior, please contact the maintainer directly.

For sensitive reports, use the contact details described in [SECURITY.md](SECURITY.md) when appropriate, or contact the maintainer privately through the same public channels listed there.

## Final Note

This project values strong opinions, direct technical feedback, and careful review.

That is fully compatible with being respectful.

Professional disagreement is welcome. Personal hostility is not.
